XSSChat lets you write your messages in HTML, define styles that persist across messages (and apply to old ones), execute Javascript directly from the chat interface, and send that Javascript to others in the room for execution. This means you can do anything you want to the site or anybody in your chat room, with no restrictions. Note that JS must be prepended with a "!" to tell the app that it should execute it as code instead of rendering it on the page as a message.

Messages are handled by a server hosted with Railway at https://connect.xsschat.com using Socket.IO. This version runs in a Node environment but does not use Express like the previous version. There is no encryption beyond that of HTTPS.

Why does the hero section say "new" ?

XSSChat (more formally, XSSChat Reloaded) is a rewrite of an earlier (not very good) chat app whose entire purpose was to allow and facilitate XSS attacks (and not get caught doing so in class). It was really not pleasant to use, so this is the improved version.
